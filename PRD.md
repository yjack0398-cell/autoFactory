请帮我创建一个内容监控工具的前端原型页面。该工具用于按分类管理不同关键词/博主的内容监控任务，支持多平台、多关键词、多博主的监控配置，并通过 AI 对采集到的热门内容进行分析，产出选题报告与洞察建议。系统设置完成后，每天定时运行一次。

二、整体结构
1. 监控分类管理（顶层）
支持创建和管理多个监控分类，例如：
Claude Code 的选题监控
Vibe Coding 的选题监控
每个分类相互独立，拥有各自的监控配置和内容数据
2. 分类内部 Tab 页（每个分类下包含三个 Tab）
Tab 1：内容
Tab 2：选题分析与报告
Tab 3：监控设置

三、各 Tab 详细需求
Tab 1：内容
展示该分类下各平台采集到的所有内容。
平台筛选：不使用下拉选择框，而是将所有平台以平铺标签/按钮的形式直接展示，用户可以直接点选切换平台。
时间线浏览：支持按时间线查看每一天采集了哪些内容。
时间选择交互需要更直观友好，避免让用户手动输入或在日历中盲选。建议采用更优的交互方式，例如：可视化时间轴滑块、日期卡片横向滚动、带有数据标记的日历热力图等（请给出你认为最佳的交互方案）。
假数据：请预先填充一些假数据，方便我查看和调整展示形式。
Tab 2：选题分析与报告
将"选题报告"和"选题分析"合并为一个 Tab。
核心功能
使用 AI（OpenAI ChatGPT）对热门内容（如当天热度最高的前 10 个内容）进行分析，产出分析报告和选题洞察。
报告内容包括：选题建议、前一天用户关注的热点等。
查看与筛选维度
(a) 时间线维度：支持按时间线查看每一天的选题报告。
时间选择交互同样需要直观友好，用户可能第一时间不知道该选哪天，因此需要更主动引导的交互方式（例如：默认展示最新报告、日期列表带预览摘要、可直接点选的日期卡片等——请给出你认为最佳的方案）。
(b) 选题维度：支持筛选查看最近一段时间内的所有选题汇总。
选题内容字段
每个选题保留以下信息：
方向标题
选题简介：包括为什么做这个选题、该选题的爆点及增长空间在哪里
Tab 3：监控设置
配置该分类下的监控目标。
支持选择/管理：
监控平台（如抖音、小红书、微博、B站等）
对标关键词
对标博主/账号
即：用户可以在特定分类下，灵活配置需要监控的关键词或对标账号。

四、设计要求
整体风格简洁专业，适合内容运营人员日常使用。
交互设计注重直观性，减少用户思考成本（尤其是时间选择、平台筛选等高频操作）。
